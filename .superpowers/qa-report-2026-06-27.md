# Visual QA — tickertracker.info — 2026-06-27

Live site, desktop 1440px, dark + light themes. (Mobile + remaining views: spot-checked.)

## Findings

### MAJOR

**M1 — Per-symbol news is broken site-wide; panel stuck on "Loading news…" forever.**
- `/api/news?sym=<X>` returns **0 items for every ticker** tested (AAPL, NVDA, TSLA, AMZN, MSFT). Market news (`?market=1`) works (12 items).
- Root cause: Finnhub FREE tier returns almost no `company-news` (even a 30-day window gives ~1 item for AAPL), and the BUG-018 URL filter then drops the `finnhub.io/api`/empty-URL ones → 0.
- Frontend bug compounds it: `NewsCard.tsx:74` shows "Loading news…" whenever `items.length === 0` — it cannot tell "still fetching" from "fetched, empty", so it shows the loading text permanently.
- FIX: (a) backend — when company-news is empty for a symbol, fall back to general market news so the panel always has content; widen window to ~30d as a minor help. (b) frontend — track a real loaded/loading state so an empty result shows "No recent news" not "Loading…".

**M2 — Header wordmark overlaps the "Strategy" nav item at desktop widths.**
- The centered "Ticker Tracker" wordmark visually collides with the last nav item ("Strategy") at 1440px (worse at tablet). Reads as "Strateg​y Ticker Tracker".
- FIX: hide the centered wordmark below a wider breakpoint, or reserve space / reduce nav so they don't collide. (Known as BUG-009; now also at 1440.)

### MINOR / POLISH

**P1 — Volume shows "—" in Key Statistics** (Finnhub /quote has no volume). Honest, but a real volume source (Yahoo quote already returns it) would fill it. Low priority.

**P2 — Light-mode page `<body>` edge stays dark** (index.css static dark bg). Not visible when the app fills the viewport (verified — light mode looks clean), but would show on overscroll. Cheap fix: body bg → var(--bg) or move bg to root.

## What looks GOOD (verified)
- Dashboard (dark): real prices (AAPL $283.82, prev_close populated), candle chart, per-card sparklines, key stats, analyst ratings (68 analysts, price targets), earnings panel — all render well.
- **Light theme looks genuinely good** — clean white surfaces, readable text, chart/sparkline colors distinct. The var() migration succeeded.
- "As of" timestamps present on Key Statistics. Console: 0 errors.
